## 2. The Ingestion Decision Tree (Checklist)

Before migrating or drafting content for `nealmcspadden.com`, run the concept through the following checklist. A piece of content **must receive a "YES" on all relevant checks** to be published.

- [ ] **AEO/SEO Anomaly Check:** Does this content contain original data, structured definitions, or mathematical models that an AI engine would want to reference and cite (high "information gain")?
- [ ] **First-Person Tone Check:** Is this written in a raw, authentic, operator-first voice (using the "Flaws Forward" paradigm) rather than a corporate marketing style?
- [ ] **Tax Sherpa / Entity Alignment:** Does this content directly or indirectly validate the core capability of Tax Sherpa, Resilient Roots, or another endorsed venture? Does it have a clear CTA redirecting to the endorsed brand?
- [ ] **No Tactical Drift:** Is this content too tactical? (e.g., *"How to configure QuickBooks settings"* belongs on a company support docs database or tactical blog, not here. *"The Death of Spreadsheets"* belongs here.)
- [ ] **Clean Data Constraint:** Have all client-specific PII, sensitive numbers, or restricted financial data been redacted or anonymized?

## 4. Operational Ingestion Action Plan

When importing a P1/P2 framework from the private OS (`neal-os`) to the website (`nealmcspadden.com`), execute the following execution cycle:

### Step 1: Structural Remapping
Convert the raw markdown structure into Astro elements. 
* Add a sidebar **Index Map** navigation block.
* Define a clear, italicized **BLUF Summary** at the top of the body inside a distinct visual box.
* Map operational numbers, benchmarks, or tiers into responsive HTML tables.

### Step 2: The Endorsement Hook
Integrate a clear call-to-action (CTA) pointing to the relevant business. For example, if publishing `ClearPath`, end the framework by explaining how it integrates with the **Tax Sherpa Board Meeting** protocol under Section 280A(g) (Augusta Rule), and place a high-contrast conversion button at the bottom.

### Step 3: AEO Integration (Schema)
Insert a JSON-LD structured script tag in the `<head>` of the page layout:
```json
<script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "WebPage",
    "name": "Framework Name",
    "description": "Short Description",
    "speakable": {
      "@type": "SpeakableSpecification",
      "cssSelector": [".aeo-speakable-title", ".aeo-speakable-bluf"]
    }
  }
</script>
```
Ensure classes `.aeo-speakable-title` and `.aeo-speakable-bluf` are applied to the title heading and the BLUF text respectively.
